# 🍎 תיעוד אוטומטי של Apple Pay באייפון — הקמה ללקוח

מדריך חד-פעמי. אחרי ההקמה, **כל תשלום Apple Pay בחנות נרשם לבד** בטאב תיעוד
ההוצאות — כולל התראה במסך הנעילה עם סטטוס התקציב.

העמוד `https://app.orimipuy.com/connect` מציג את אותם שלבים כאשף מקוצר —
המדריך הזה הוא הגרסה המפורטת (ליועץ / לתקלות).

## דרישות
- iPhone עם iOS 17 ומעלה (השמות כאן לפי iOS 26).
- Apple Pay מוגדר עם כרטיס.
- המייל של הלקוח מאושר ב-allowlist (invite-only).

## מה זה תופס / לא תופס
- ✅ תשלומי **Apple Pay פיזיים בחנות** (קירוב NFC).
- ❌ **לא** קניות אונליין, ולא כרטיס פיזי רגיל (בלי Apple Pay).

## ההקמה — 4 שלבים (~5 דקות)

### שלב 1 — טוקן
ב-iPhone, בספארי: `https://app.orimipuy.com/connect` → התחברות →
**"📋 העתק טוקן"**.

### שלב 2 — הוספת הקיצור והדבקת הטוקן
1. באותו עמוד: **"📲 הוסף את הקיצור"** — נוסף הקיצור **THE HOME ECONOMIST 1**
   (קישור iCloud: `https://www.icloud.com/shortcuts/1330de3c3651478a8c5dd939f4b1a53c`).
2. באפליקציית **קיצורי דרך**: לחיצה **ארוכה** על THE HOME ECONOMIST 1 →
   **"עריכה"** → בתיבת המלל מחק את «הדבק כאן טוקן» → **הדבק את הטוקן** → סיום.

### שלב 3 — אוטומציית הארנק (החלק שאפל מחייבת לבנות ידנית)
1. קיצורי דרך → לשונית **"פעולות אוטומטיות"** (האמצעית) → **＋**.
2. גלול ובחר **"ארנק"** (ב-iOS ישן: "עסקה"/Transaction) → **"כאשר אני מקיש"**
   → בחר כרטיס/ים → **"הפעל מיד"** → הבא.
3. הוסף שתי פעולות (דרך "חיפוש פעולות" למטה — לא מהכפתור הירוק שבתוך פעולה):
   - **"מלל"** — בתיבה: «בחירת משתנה» → **"קלט של קיצור"** → הקש על הצ'יפ
     שנוסף → בחר **"כמות"** → הקש **מקש רווח** → שוב «בחירת משתנה» →
     "קלט של קיצור" → הקש על הצ'יפ → **"בית עסק"**.
     (בחירת המאפיינים עובדת רק כאן, בהקשר האוטומציה — לא בתוך קיצור עצמאי.)
   - **"הפעל קיצור דרך"** — בחר **THE HOME ECONOMIST 1**.
4. סיום. ⚠️ אם iOS שואל "לאפשר הפעלה בלי אישור?" — לאשר (אחרת כל תשלום ידרוש הקשה).

### שלב 4 — האפליקציה למסך הבית
בספארי, ב-`app.orimipuy.com`: כפתור שיתוף → **"הוספה למסך הבית"** → הוסף.

### בדיקה
תשלום Apple Pay בחנות → תוך שניות: התראת מסך-נעילה "נרשם: [עסק] · ₪[סכום]"
עם שורת תקציב, וההוצאה בטאב תיעוד ההוצאות, מקוטלגת. ✅

## איך זה עובד (ליועץ)

```
אוטומציית ארנק (על כל מכשיר, ידנית):
  קבלת עסקה בתור קלט
  מלל:  [כמות] [בית עסק]        ← המאפיינים זמינים רק בהקשר אוטומציה
  הפעל קיצור דרך → THE HOME ECONOMIST 1 (הטקסט עובר כקלט)

הקיצור המשותף (מיובא בהקשה, כל הלוגיקה בפנים):
  מלל: <הטוקן של הלקוח>
  קבלת התוכן של https://app.orimipuy.com/api/transaction
      POST JSON { token: מלל, merchant: קלט של קיצור }   ← אין שדה amount!
  קבלת ערך מהמילון notify.title ← תוכן כתובת האינטרנט
  קבלת ערך מהמילון notify.body  ← תוכן כתובת האינטרנט
  הצגת העדכון (כותרת=title, גוף=body)   ← "הצגת העדכון" = ההתראה האמיתית;
                                            "הצגת התראה" = חלונית חוסמת, לא להשתמש!
```

- השרת (`/api/transaction` → `extractFromRaw`) מפרק את הטקסט "[כמות] [בית עסק]"
  לסכום + שם עסק, כולל ניקוי תווי bidi נסתרים ש-iOS מוסיף (קומיט b346633).
- תגובת השרת כוללת `notify{title,body,warn}` עם סטטוס תקציב — אותו טקסט
  שמוצג באנדרואיד.
- הרצה ידנית (▶) של הקיצור תמיד תיגמר בשגיאה על מפתח notify — צפוי: בלי עסקה
  אמיתית התשובה היא `{"error":"bad merchant"}` בלי notify. לא באג.
- כשל רשת ברגע התשלום = ההוצאה אובדת (אין retry בקיצורי דרך); iOS מציג באנר
  "ההפעלה האוטומטית נכשלה" — זה הסימן להוסיף ידנית.

## תקלות מוכרות
- **‎-1005 "החיבור לרשת אבד" בכל הרצה** — פעולת "קבלת התוכן" הושחתה מעריכה
  חוזרת. מוחקים רק אותה ובונים מחדש. לא לחפש בעיות רשת/טוקן.
- **התראה מופיעה רק כשהטלפון פתוח / דורשת אישור** — שולבה "הצגת התראה"
  (החוסמת) במקום "הצגת העדכון". להחליף.
- **הכותרת והגוף של ההתראה זהים** — שדה הכותרת קיבל את ערך המילון של body;
  לחווט לערך של הריבוע הראשון (notify.title).
- **"bad merchant" בהרצה ידנית** — תקין (אין קלט). רק תשלום אמיתי בודק.
