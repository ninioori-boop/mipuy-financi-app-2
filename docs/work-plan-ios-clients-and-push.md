# תוכנית עבודה — לקוחות אייפון + התראות ממותגות

_נוצר: 2026-07-14 · אושר ע"י אורי. שלוש מטרות: (1) לקוח אייפון מתחבר בפשטות,
(2) התראות קליטה אמינות, (3) התראות עם מיתוג האפליקציה._

## עקרונות
- מגבלות אפל שאי אפשר לעקוף: אוטומציית ארנק חייבת להיבנות ידנית על כל מכשיר
  (אין שיתוף/יצירה אוטומטית); התראה מקיצורי דרך תמיד עם אייקון "קיצורי דרך".
- לכן: מצמצמים את חלק הלקוח למינימום האפשרי (קיצור משותף + אוטומציה של שורה
  אחת), והמיתוג מגיע מתשתית push של האפליקציה עצמה.
- כל שינויי הווב/שרת — תוספות בלבד (additive), בלי לגעת בזרימות חיות, הסבר
  לפני נגיעה בצנרת משותפת.

---

## שלב 1 — קיצור Mipuy משותף (אורי על הטלפון + אני בווב)

**מטרה:** כל הלוגיקה בקיצור אחד שהלקוח מוסיף בהקשה; האוטומציה שלו = טריגר
ארנק + "הפעל את Mipuy" בלבד (~30 שניות).

- [ ] **1a. בניית הקיצור על הטלפון של אורי** — לשונית "קיצורי דרך" → Mipuy:
      מלל (טוקן) → קבלת התוכן של `https://app.orimipuy.com/api/transaction`
      (POST, JSON: token=מלל, merchant=קלט של קיצור→בית עסק, amount=קלט של
      קיצור→כמות) → קבלת ערך מהמילון `notify.title` → `notify.body` (שניהם על
      "תוכן של כתובת אינטרנט") → **הצגת התראה מהסוג הלא-חוסם** (יש "השמעת
      צליל" + "כותרת"; אין "הצגת כפתור הביטול").
      ⚠️ נקודת סיכון: אם "בית עסק"/"כמות" לא מופיעים בבחירת מאפיין על "קלט של
      קיצור" בקיצור עצמאי — תוכנית ב׳: האוטומציה בונה מלל "[כמות] [בית עסק]"
      (double-selection עובד בהקשר אוטומציה) ומעבירה לקיצור; השרת מפרק
      (extractFromRaw קיים ועובד).
- [ ] **1b. חיווט האוטומציה של אורי** — מחיקת הפעולות הפנימיות → פעולה אחת
      "הפעל קיצור דרך" → Mipuy.
- [ ] **1c. אימות בתשלום אמיתי** — נרשם + התראת באנר (גם בנעילה). בדיקת שרת
      ב-`npx tsx scripts/check-inbox.ts`.
- [ ] **1d. שיתוף** — החלפת הטוקן ב"הדבק כאן טוקן" → שיתוף → קישור iCloud →
      אורי שולח לי.
- [ ] **1e. עדכון /connect (אני)** — החלפת `SHORTCUT_ICLOUD_URL` + עדכון נוסח
      האשף לשמות iOS 26 ("פעולות אוטומטיות"/"ארנק"/"הפעל מיד") + עדכון
      `docs/ios-shortcut-setup.md`. פריסה ל-main באישור אורי.
- [ ] **1f. עדן — הלקוחה הראשונה** (כבר מאושרת ב-allowlist): התחברות →
      /connect: העתק טוקן → הוסף קיצור → הדבק טוקן → אוטומציה קצרה → הוספה
      למסך הבית. תשלום בדיקה.

## שלב 2 — התראות ממותגות (Web Push) — הוקדם לפני ה-Play Store לבקשת אורי

**מטרה:** התראה עם הלוגו והשם "הכלכלן של הבית" על כל קליטה, מהאפליקציה
המותקנת (iOS 16.4+ PWA; עובד גם בדפדפני אנדרואיד/דסקטופ). מחליפה בהדרגה את
התראת קיצורי הדרך.

- [ ] **2a. שרת** — זוג מפתחות VAPID (env: `VAPID_PUBLIC_KEY`,
      `VAPID_PRIVATE_KEY`, `NEXT_PUBLIC_VAPID_PUBLIC_KEY` לציבורי בלבד) +
      route חדש `/api/push-subscribe` (אימות Firebase token; שומר subscription
      ב-Firestore דרך admin SDK — בלי שינוי rules) + ספריית web-push.
- [ ] **2b. ווב** — service worker (`public/sw.js`): קבלת push + הצגת התראה +
      פתיחת האפליקציה בהקשה; כרטיס "הפעל התראות" (חייב מחוות משתמש ב-iOS;
      מוצג רק ב-PWA מותקן/דפדפן תומך).
- [ ] **2c. שליחה** — ב-`/api/transaction`, אחרי קליטה מוצלחת: שליחת push
      best-effort לכל ה-subscriptions של המשתמש עם `buildNotify` (הטקסט כבר
      קיים). כשל שליחה לעולם לא מפיל את הקליטה. ניקוי subscriptions מתים (410).
      שינוי בצנרת חיה → הסבר + אישור לפני פריסה.
- [ ] **2d. כפילויות** — משתמשי אפליקציית האנדרואיד מקבלים התראה מקומית מהשל
      הנייטיבי ולא נרשמים ל-push (ה-WebView לא נרשם) — אין כפל טבעי. לקוח
      אייפון שנרשם ל-push: מסירים את פעולת ההתראה מהקיצור שלו (או משאירים —
      לבחירת אורי אחרי שיראה).
- [ ] **2e. בדיקות** — האייפון של אורי (PWA מותקן) → עדן → עדכון מסמכי ההדרכה.

## שלב 3 — אחרי זה (לא בתוכנית הזאת)
- השקת Play Store (המסלול הקיים, לא חסום ע"י העבודה כאן).
- App Store — יעד עתידי של אורי (99$/שנה, מעטפת iOS עם ערך נייטיבי, CI על
  macOS, ביקורת אפל). נבחן אחרי שה-push חי והמודל מוכח.
