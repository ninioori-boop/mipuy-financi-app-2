המשתמש (אורי, יועץ פיננסי) רוצה להוסיף פגישה/ות ליומן הפגישות שלו.

הקלט: **$ARGUMENTS**

הקלט הוא פגישה אחת או כמה (שורה לכל פגישה, או מופרדות בפסיק). כל פגישה בפורמט חופשי "שם-סוג" / "סוג-שם" — למשל `אופק-מיפוי`, `היכרות-דני`, `תקציב-עדן 21/6`.

בצע בדיוק את הצעדים הבאים:

1. **פענח כל פגישה** — חלץ ממנה: שם לקוח, סוג פגישה, ותאריך (אם צוין).

2. **מפה את הסוג** ל-`type` לפי הטבלה (אלה המפתחות הקיימים ב-`invoicing/meetings.json` תחת `types`):

   | מה שאורי כותב | type | מחיר |
   |---------------|------|------|
   | מיפוי | `מיפוי` | 250 ₪ |
   | תקציב | `תקציב` | 250 ₪ |
   | תוכנית / תוכנית כלכלית | `תוכנית` | 250 ₪ |
   | בקרה | `בקרה` | 250 ₪ |
   | היכרות / הכרות | `היכרות` | 100 ₪ |
   | השקעה / השקעה ראשונה / פגישת השקעה | `השקעה ראשונה` | 90 ₪ |

   אם הסוג לא ברור או לא מתאים לאף שורה — **עצור ושאל את אורי** לפני שתוסיף.

3. **תאריך**: אם צוין (כמו `18/6`) המר ל-`YYYY-MM-DD` (שנת 2026 אם לא נכתבה שנה). אם לא צוין — השתמש בתאריך של היום.

4. **חיוב**: כל פגישה היא **לחיוב** כברירת מחדל — גם פגישה ש"לא עלתה". הערה אף פעם לא מורידה תשלום. הוסף `"billable": false` **רק** אם אורי אמר במפורש לא לחייב על פגישה מסוימת.

5. **הוסף ל-`invoicing/meetings.json`**: קרא את הקובץ, והוסף למערך `meetings` אובייקט לכל פגישה חדשה, בפורמט:
   ```json
   { "date": "YYYY-MM-DD", "client": "<שם הלקוח>", "type": "<key>", "notes": "<הערה אם יש, אחרת ריק>" }
   ```
   הקפד על JSON תקין (פסיק בין אובייקטים, בלי פסיק אחרי האחרון).

6. **בנה מחדש את האקסל**:
   ```
   node invoicing/build-sheet.js "invoicing/יומן-פגישות-2026.xlsx"
   ```
   אם חוזרת שגיאת `EBUSY` — הקובץ פתוח ב-Excel. בקש מאורי לסגור אותו, ואז הרץ שוב.

7. **דווח בקצרה בעברית**: אילו פגישות נוספו (שם + סוג + מחיר), ומה **הסה״כ המעודכן של אותו חודש**. אם נוספו פגישות לחודש שעדיין אין לו טאב — צוין שהטאב נוצר. הזכר לאורי שאם הוא רוצה לרענן גם את הגיליון בדרייב — שיעלה מחדש את הקובץ.

אל תעשה שום דבר נוסף — זו פעולה ממוקדת אחת.
